## ⚠️ Contraintes critiques — erreurs déjà vécues en V1.2

### 1. Freezed — toujours `abstract class`
```dart
// ✅ CORRECT
@freezed
abstract class MyModel with _$MyModel {
  const factory MyModel({...}) = _MyModel;
}

// ❌ INTERDIT — génère une erreur de mixin
@freezed
class MyModel with _$MyModel { ... }
```
Après toute modification d'un modèle : `dart run build_runner build --delete-conflicting-outputs`

### 2. AppColors — tokens existants uniquement
```dart
// ✅ Tokens valides
context.colors.bg        // fond principal
context.colors.bg2       // fond secondaire (cartes)
context.colors.ink       // texte principal
context.colors.muted     // texte secondaire
context.colors.accent    // couleur accent (or/violet)
context.colors.accent2   // accent secondaire
context.colors.line      // séparateurs / bordures
context.colors.error     // rouge erreur

// ❌ N'existent PAS — ne pas utiliser
// context.colors.surface
// context.colors.text
// context.colors.onPrimary
// context.colors.primary
```

### 3. AppTypography — tokens existants uniquement
```dart
// ✅ Tokens valides
context.typo.displayLarge
context.typo.displayMedium
context.typo.headline
context.typo.bodyLarge
context.typo.body
context.typo.caption
context.typo.overline
context.typo.button

// ❌ N'existent PAS
// context.typo.labelLarge / labelMedium / labelSmall
// context.typo.titleMedium
```

### 4. L10n — ne jamais éditer les .dart générés
```
Source de vérité : l10n/intl_fr.arb  +  l10n/intl_ar.arb
Après ajout de clés : flutter gen-l10n
❌ Ne jamais éditer lib/l10n/app_localizations*.dart directement
   (ces fichiers sont écrasés à chaque gen-l10n)
```

### 5. Navigation
```dart
context.push('/route')   // navigation
context.pop()            // retour
// Toujours utiliser _fadeSlide() dans app_router.dart pour les pageBuilder
```

### 6. State management
```dart
// ref.watch() dans build() uniquement
// ref.read() dans les callbacks/handlers uniquement
// Riverpod uniquement — pas Provider, pas setState sur ConsumerWidget
```

### 7. HiveSource — pattern de persistance existant
Le projet utilise **une seule box Hive** `'settings'` avec un blob JSON manuel.
Pas de TypeAdapter, pas de box séparée. Pour ajouter des champs :
- Ajouter dans `UserState` (freezed)
- Ajouter la sérialisation dans `HiveSource._toMap()` et `HiveSource._fromMap()`
- Re-générer le freezed

## Périmètre V1.3

### Feature 1 — SRS Leitner (Révision espacée)

Algorithme à 3 niveaux par nom :
- **0 = Nouveau** (défaut)
- **1 = En cours**
- **2 = Maîtrisé**

Règles :
- "Je connais" → monte d'un niveau (max 2)
- "Encore flou" → redescend d'un niveau (min 0)
- Session de révision : présente d'abord les niveau 0, puis niveau 1
- Connexions avec l'existant :
  - Tafakkur complet (`isComplete == true`) → `levelUp(nameNumber)`
  - Bonne réponse QCM → `levelUp(nameNumber)`
  - "Je connais" en Flashcard → `levelUp(nameNumber)`

### Feature 2 — Parcours Thématiques

8 parcours prédéfinis, chacun couvrant 6–10 noms liés par thème éditorial.
Données dans `assets/data/parcours.json`.
Navigation séquentielle fiche par fiche, marquable comme complété.

Thèmes suggérés (à affiner selon les catégories réelles dans names.json) :
1. Noms de la Miséricorde
2. Noms de la Gloire et Majesté
3. Noms du Savoir Absolu
4. Noms du Guide et de la Protection
5. Noms de la Beauté
6. Noms de la Création
7. Noms du Jugement
8. Noms de la Prophétie

### Feature 3 — Mode de découverte

Écran d'entrée avant toute session d'étude, proposant :
- **Parcours thématique** → liste des 8 parcours
- **Aléatoire** → mélange de noms non maîtrisés
- **Manuel** → filtre par catégorie

Choix persisté dans UserState (`studyMode`), re-choisissable depuis cet écran.

### Architecture générique multi-deck (investissement V1.4)

Les features SRS et Parcours sont conçues pour fonctionner sur n'importe quel deck de données, pas seulement les 201 noms. En V1.4, ajouter les 99 Noms d'Allah = ajouter un JSON + déclarer un `LearningDeck`. Aucune modification du core.

```dart
'leitnerBoxes': s.leitnerBoxes.map((k, v) => MapEntry(k.toString(), v)),
'completedParcours': s.completedParcours.toList(),
```

Ajouter dans `HiveSource._fromMap()` :
```dart
final leitnerRaw = m['leitnerBoxes'] as Map<String, dynamic>? ?? {};
final leitnerBoxes = leitnerRaw.map((k, v) => MapEntry(int.parse(k), v as int));
final completedParcoursList = m['completedParcours'] as List<dynamic>? ?? [];
```
